Glossary

Claude's Xcode 26 Swift Bible

App Clip

A small, lightweight piece of your app (under 15MB) that someone can use instantly without installing the full app. The user scans a QR code, taps an NFC tag, or clicks a link, and a functional slice of your app launches on the spot — no App Store download, no install. Think of it like scanning a code at a restaurant and immediately getting an ordering screen. If the user likes it, they can install the full app from there. Mostly used by businesses with physical-world touchpoints like restaurants, parking meters, and scooter rentals. Most solo developers don't need one.

ARC (Automatic Reference Counting)

Swift's memory management system. It counts how many things are using an object. When nothing needs it anymore, ARC cleans it up automatically. No manual memory allocation or deallocation needed.

Class

A container for data and behavior, like a struct, but with one key difference: a class is a reference type. When you pass a class instance to a function or assign it to another variable, everyone shares the same object — they're all pointing to the same thing in memory. Change it in one place, it changes everywhere. This makes classes useful when you need shared state (like an @Observable data model that multiple views read from). The tradeoff is more complexity — you have to think about who's modifying the object and when. Structs are simpler and preferred in Swift unless you specifically need reference semantics.

Computed Property

A property that runs code every time you read it, instead of storing a fixed value. A stored property is like a box with a value sitting in it: var name = "Michael". A computed property is like asking a question — it calculates the answer fresh each time: var greeting: String { return "Hello, \(name)" }. In SwiftUI, body is a computed property because the framework needs to ask "what should this view look like right now?" every time something changes.

Compiled Language

A language where your source code gets translated into machine code before it runs. The translation happens once at build time, not every time the program runs. Swift is compiled. Python and JavaScript are interpreted — they translate on the fly, which is slower.

Git

A version control system that tracks changes to your code over time. Every save point is called a "commit." You can go back to any previous commit, branch off to try something new, and merge changes together. Created by Linus Torvalds in 2005 (the same person who created Linux). Git runs locally on your Mac — it doesn't need the internet.

GitHub

A website (github.com) that hosts Git repositories online. It's like a cloud backup for your code, but it also adds collaboration tools — pull requests, issues, wikis, and GitHub Pages for free websites. Owned by Microsoft since 2018. Your code on GitHub can be public (anyone can see it) or private (only you and people you invite). Git is the tool. GitHub is the service that stores your Git repos online.

IDE (Integrated Development Environment)

A program that combines a code editor, build tools, debugger, and other development tools into one application. Xcode is Apple's IDE.

Info.plist

A configuration file that tells the system about your app before it even launches. It's where you declare things like your app's display name, what permissions it needs (camera, microphone, location), which orientations it supports, and what URL schemes it handles. Think of it as the label on the outside of the box — the system reads it to know what's inside without opening the app. Every target has its own Info.plist because each product may need different permissions or settings.

Int (Integer)

Short for Integer. A whole number with no decimal point. Examples: 0, 42, -7, 1000000. In Swift, Int is the default type for whole numbers. It's 64-bit on modern Apple hardware, meaning it can hold numbers up to about 9.2 quintillion.

Machine Code

The lowest-level instructions that a processor (CPU) actually executes. It's binary — ones and zeros. Humans don't write machine code directly. You write Swift, the compiler translates it into machine code, and the processor runs that. This is why compiled languages run fast — the translation is already done before the app launches.

Remote Repository

A copy of your Git repository stored on a server somewhere else — usually GitHub. Your Mac has the local copy, GitHub has the remote copy. You "push" to send your changes up and "pull" to bring changes down. Think of it like saving a document to iCloud — the file exists on your Mac and in the cloud. If your Mac dies, the remote still has your code.

Repository (Repo)

A folder that Git is tracking. It contains your code plus a hidden .git folder that stores the entire history of every change ever made. You can have a local repo on your Mac, a remote repo on GitHub, or both.

Normalization

Literally: to make normal. Take a sporadic jumble and organize it so everything has a place and nothing is duplicated. The term originates from database normalization (Edgar F. Codd, IBM, 1970) — each concept gets its own table, no data is repeated across tables. In code, the same principle means each file handles one responsibility: one view per file, models in their own folder, services separate from UI. The opposite of normalization is a **monolithic** file — one giant file with everything piled together. It works until it doesn't, and then you can't find anything or change anything without breaking something else.

Persistence

Saving information so it survives after the app closes. Without persistence, everything your app knows disappears when the user quits — like a whiteboard that gets erased every night. With persistence, things like user settings, journal entries, saved documents, and favorites are written to disk and loaded back the next time the app opens. SwiftData, Core Data, UserDefaults, and saving files to the file system are all forms of persistence. CloudKit takes it further by syncing that saved information across the user's devices.

Protocol

A contract that defines what a type must be able to do, without saying how it does it. Think of it like a job description — it lists the requirements, but each person who fills the role does the work their own way. In Swift, a type "conforms" to a protocol by implementing everything the protocol requires.

Struct (Structure)

A lightweight container that bundles data and behavior together as a single unit. In Swift, most of what you build is a struct — your views, your data models, your custom types. Unlike a class, a struct is a value type: when you pass it to a function or assign it to another variable, Swift makes a copy instead of sharing the original. SwiftUI views are structs because they're meant to be cheap, disposable descriptions of what the screen should look like — SwiftUI creates and destroys them constantly.

String

Text. A sequence of characters — letters, numbers, symbols, spaces, emojis. In Swift, Strings are wrapped in double quotes: "Hello", "CryoTunes Player", "42". Note that "42" as a String is text, not a number — you can't do math with it until you convert it to an Int.

Target

A word with three meanings in Xcode. Build target: a product your project builds (an app, a test suite, a widget extension — one project can build several). Run destination: the device or simulator you run the app on (the dropdown in the Xcode toolbar). Deployment target: the minimum OS version your app supports — the floor that determines who can download it from the App Store. Context usually makes it clear which one someone means.

View

Anything that can be drawn on screen in SwiftUI. A text label is a View. A button is a View. A whole screen is a View made of smaller Views. Every View is a struct with a body property that describes what it looks like. SwiftUI calls body whenever it needs to know the current appearance, and only redraws the parts that actually changed.

View Protocol

The contract a struct must fulfill to be drawn on screen by SwiftUI. It has one requirement: a computed property called body that returns some View. When you write struct MyScreen: View, you're telling Swift "this struct can be displayed as part of the user interface." The View protocol is what connects your code to SwiftUI's rendering engine.

Type-safe

A language where the compiler checks that you're using the right kind of data in the right place. You can't accidentally treat text as a number or a number as a list. The compiler catches these mistakes before your code runs, not after.
