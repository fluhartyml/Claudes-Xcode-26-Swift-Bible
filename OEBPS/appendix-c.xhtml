Appendix C: Apple Frameworks & Kits Reference

Claude's Xcode 26 Swift Bible

Each kit listed here gets a comprehensive explanation — what it does, why you'd use it, what entitlements or permissions it requires, and which chapter covers it in detail. This appendix grows as chapters are written.

App Intents

What it does: Lets your app integrate with Siri, Shortcuts, Spotlight, and Apple Intelligence. You define "intents" — actions your app can perform — and the system surfaces them to the user through voice commands, suggestions, and the Shortcuts app.

Introduced: WWDC 2022 (iOS 16)

Replaces: SiriKit intents for new development

Entitlement required: No

Privacy description required: No (but the actions your intent triggers may require permissions)

developer.apple.com setup: No

Covered in: TBD

CloudKit

What it does: Syncs data across a user's devices via iCloud. Public databases let all users of your app share data. Private databases sync a single user's data across their iPhone, iPad, Mac, and Apple TV.

Introduced: WWDC 2014 (iOS 8)

Entitlement required: Yes — iCloud container identifier

Privacy description required: No

developer.apple.com setup: Yes — enable iCloud under App Services, create a CloudKit container

Free tier: Generous — 1GB asset storage, 100MB database, 2GB transfer per user

Covered in: TBD

Core Data

What it does: Apple's original on-device persistence framework. Object graph management with SQLite backing. Handles relationships, migrations, undo/redo, and lazy loading.

Introduced: 2005 (Mac OS X Tiger)

Status: Still supported, but SwiftData is the modern replacement

Entitlement required: No (unless syncing via CloudKit)

Covered in: Ch 15

Core Location

What it does: GPS, Wi-Fi, and Bluetooth-based location services. Current location, geofencing, heading, visit monitoring, and significant location changes.

Introduced: 2008 (iPhone OS 2)

Entitlement required: No

Privacy description required: Yes — NSLocationWhenInUseUsageDescription and/or NSLocationAlwaysAndWhenInUseUsageDescription

developer.apple.com setup: No

Covered in: TBD

Foundation Models Framework

What it does: Direct access to Apple's on-device large language models. Text generation, summarization, and other LLM tasks — entirely on-device, no internet required.

Introduced: WWDC 2025 (iOS 26)

Entitlement required: No

Privacy description required: No

developer.apple.com setup: No

Covered in: TBD

GameKit

What it does: Game Center integration — leaderboards, achievements, matchmaking, player authentication, challenges. Works across all Apple platforms.

Introduced: 2010 (iOS 4)

Entitlement required: Yes — Game Center capability

Privacy description required: No

developer.apple.com setup: Yes — configure Game Center in App Store Connect

Covered in: TBD

HealthKit

What it does: Read and write health and fitness data — steps, heart rate, sleep, nutrition, workouts, medications. Shared health data store across all apps that have permission.

Introduced: WWDC 2014 (iOS 8)

Entitlement required: Yes — HealthKit capability

Privacy description required: Yes — NSHealthShareUsageDescription (read) and NSHealthUpdateUsageDescription (write)

developer.apple.com setup: No

Platforms: iPhone and Apple Watch only (not iPad, not Mac)

Covered in: TBD

HomeKit

What it does: Smart home device control. Lights, locks, thermostats, cameras. Bridges to Matter and Thread protocols. Automations and scenes.

Introduced: WWDC 2014 (iOS 8)

Entitlement required: Yes — HomeKit capability

Privacy description required: Yes — NSHomeKitUsageDescription

developer.apple.com setup: No

Covered in: TBD

MapKit

What it does: Embed Apple Maps in your app. Annotations, overlays, directions, search, Look Around (street-level imagery). Works on all Apple platforms.

Introduced: 2009 (iPhone OS 3)

Entitlement required: No

Privacy description required: Only if using Core Location for the user's position

developer.apple.com setup: No

Covered in: TBD

MusicKit

What it does: Access the Apple Music catalog (80+ million songs) and the user's personal music library. Search, playback, playlists, album art, now playing info.

Introduced: WWDC 2021 (iOS 15) — Swift-native rewrite of the older MediaPlayer/StoreKit approach

Entitlement required: Yes — MusicKit capability

Privacy description required: Yes — NSAppleMusicUsageDescription

developer.apple.com setup: Yes — must enable MusicKit under App Services in Certificates, Identifiers & Profiles. The Xcode checkbox alone is not enough.

Covered in: TBD

ShazamKit

What it does: Audio recognition. Match audio against Shazam's catalog (identify songs) or against your own custom audio signatures (match specific audio content).

Introduced: WWDC 2021 (iOS 15)

Entitlement required: No

Privacy description required: Yes — NSMicrophoneUsageDescription (if listening through the mic)

developer.apple.com setup: Yes — enable ShazamKit under App Services

Covered in: TBD

SiriKit

What it does: Voice assistant integration through predefined intent domains — messaging, payments, workouts, ride booking, photo search, and more. Your app handles Siri requests in specific categories.

Introduced: WWDC 2016 (iOS 10)

Status: Still available, but App Intents is the modern replacement for new development

Entitlement required: Yes — Siri capability

Privacy description required: Yes — NSSiriUsageDescription

developer.apple.com setup: No

Covered in: TBD

See also: App Intents

SpriteKit

What it does: 2D game engine. Sprites, physics simulation, particle effects, scene transitions, tile maps. Runs on all Apple platforms including Apple TV.

Introduced: WWDC 2013 (iOS 7)

Entitlement required: No

Privacy description required: No

developer.apple.com setup: No

Covered in: TBD

StoreKit

What it does: In-app purchases, subscriptions, and transaction management. StoreKit 2 (introduced WWDC 2021) is the modern async/await API. Test purchases locally with StoreKit configuration files in Xcode — no App Store Connect setup needed for testing.

Introduced: 2009 (iPhone OS 3); StoreKit 2 at WWDC 2021

Entitlement required: Yes — In-App Purchase capability

Privacy description required: No

developer.apple.com setup: Yes — configure products in App Store Connect

Covered in: TBD

SwiftData

What it does: Apple's modern persistence framework. Define models with the @Model macro, query with #Predicate, and let SwiftData handle the SQLite database underneath. Integrates natively with SwiftUI.

Introduced: WWDC 2023 (iOS 17)

Replaces: Core Data for new projects

Entitlement required: No (unless syncing via CloudKit)

Privacy description required: No

developer.apple.com setup: No

Covered in: Ch 15

WeatherKit

What it does: Current weather conditions, hourly and daily forecasts, severe weather alerts, historical weather data. Powered by Apple Weather.

Introduced: WWDC 2022 (iOS 16)

Entitlement required: Yes — WeatherKit capability

Privacy description required: No (but Core Location permission is needed to get weather for the user's location)

developer.apple.com setup: Yes — must enable WeatherKit under App Services. The Xcode checkbox alone is not enough.

Free tier: 500,000 API calls per month included with Apple Developer Program membership

Attribution required: Yes — Apple requires a Weather attribution link in your app

Covered in: TBD

WebKit

What it does: Embed web content in your app. WKWebView (UIKit) or the new SwiftUI WebView (introduced WWDC 2025). Same rendering engine as Safari. JavaScript evaluation, navigation delegation, cookie management.

Introduced: 2003 (Safari 1.0); WKWebView in 2014; SwiftUI WebView in 2025

Entitlement required: No

Privacy description required: No

developer.apple.com setup: No

Covered in: TBD

WidgetKit

What it does: Home Screen, Lock Screen, and StandBy widgets. Timeline-based — your app provides data snapshots at intervals, the system displays them. Interactive widgets (buttons, toggles) added in iOS 17.

Introduced: WWDC 2020 (iOS 14); interactive widgets WWDC 2023

Entitlement required: No (just add a Widget Extension target)

Privacy description required: No

developer.apple.com setup: No

Covered in: TBD

Writing Tools

What it does: System-level text rewriting, proofreading, and summarization. Automatically available in any standard text field (UITextField, UITextView, SwiftUI TextField/TextEditor). Part of Apple Intelligence.

Introduced: WWDC 2024 (iOS 18)

Entitlement required: No

Privacy description required: No

developer.apple.com setup: No

Opt-out: You can disable Writing Tools on specific text fields if needed (e.g., password fields)

Covered in: TBD
